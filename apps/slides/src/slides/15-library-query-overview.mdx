- NOT really a "fetching" library. It handles all of your promises, and async logic around that.
- Gives you loading and error state management for free
- caching, request deduping, background refetching, garbage collection
- You might not think that caching is a big deal until you use an app with and without it. It's night and day.
- Structural sharing + memoized results = fewer re-renders
- You'll have a lot less code to write with way less bugs


```tsx
import { useQuery } from "@tanstack/react-query"

function Todos() {
  const { data, isPending, error } = useQuery({
    queryKey: ["todos"],
    queryFn: getTodos,
  })

  if (isPending) return "Loading..."
  if (error) return "Error: " + error.message
  return data.map((t) => <li>{t.title}</li>)
}
```
