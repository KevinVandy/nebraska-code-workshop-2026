## RSC Pros & Cons

- ✅ Less JavaScript shipped — server components stay on the server
- ✅ Async components — `await` your data right in the component
- ✅ Streaming means fast first content without waiting for everything
- ❌ Requires a framework runtime (Next.js, or roll your own bundler config)
- ❌ New mental model — server vs client boundaries trip everyone up
- ❌ Ecosystem still settling — libraries, caching, and patterns in flux
